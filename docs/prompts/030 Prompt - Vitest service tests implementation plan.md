Twoim zadaniem jest przeanalizowanie powyższych materiałów i stworzenie kompletnego planu implementacji testów dla tego serwisu. Plan powinien zawierać:

1. **Analizę serwisu** - krótki opis funkcjonalności serwisu i jego kluczowych metod
2. **Strategię testowania** - ogólne podejście do testowania tego serwisu
3. **Szczegółowy plan testów** - dla każdej metody/funkcjonalności serwisu:
   - Nazwa testu
   - Opis tego co test sprawdza
   - Kroki testowe (Arrange, Act, Assert)
   - Wymagane mocki/stuby
   - Oczekiwane rezultaty
4. **Konfigurację testową** - niezbędne importy, setupy, konfiguracje Vitest
5. **Przypadki brzegowe** - testy dla nietypowych sytuacji, błędów, walidacji
6. **Pokrycie kodu** - wskazanie jakie części kodu będą pokryte testami

Pamiętaj o:
- Stosowaniu zasad Vitest podanych w regułach
- Tworzeniu testów jednostkowych, a nie integracyjnych
- Mockowania zależności zewnętrznych
- Testowaniu zarówno pozytywnych jak i negatywnych scenariuszy
- Używaniu opisowych nazw testów w języku polskim
- Grupowaniu powiązanych testów w bloki describe()

Wynik umieść w formacie Markdown i strukturyzuj go czytelnie z odpowiednimi nagłówkami, listami i blokami kodu. Wszystkie opisy, komentarze i nazwy testów mają być w języku polskim.

Twoja odpowiedź powinna być gotowym plikiem plan-testów-serwisu.md, który programista może bezpośrednio wykorzystać do implementacji testów.
