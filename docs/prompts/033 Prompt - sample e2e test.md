1. Przeanalizuj PRD, aby zidentyfikować najbardziej podstawową funkcjonalność aplikacji, która powinna być przetestowana w pierwszej kolejności
2. Sprawdź stos technologiczny, aby określić odpowiednie narzędzia i framework do testów e2e
3. Uwzględnij strategię testów e2e przy projektowaniu testu

6. Opisz krok po kroku, jak uruchomić ten test

Pamiętaj, że test powinien być:
- Jak najprostszy (testować tylko jedną, podstawową funkcjonalność)
- Łatwy do zrozumienia i utrzymania

- Zgodny ze strategią testów e2e
